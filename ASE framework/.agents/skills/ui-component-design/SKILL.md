# UI Component Design

## Objective

Create and customize UI components that look intentionally designed by a human — not generated by AI. Combine shadcn/ui primitives with custom Tailwind variants, Lucide iconography, and layered visual treatments that eliminate every trace of "vibe-coded" aesthetics.

## When to Use This Skill

- Building a new UI component (button, card, dialog, dashboard section, etc.)
- Restyling an existing component that looks too "template-y"
- Running a vibe-code audit on a page or set of components
- Adding icons to a component or page
- Extending shadcn/ui with custom variants or brand tokens
- Creating a bento grid, asymmetric layout, or non-standard composition

## Input Specification

| Input | Type | Required | Description |
|-------|------|----------|-------------|
| component_name | string | Yes | Name of the component to build or restyle |
| target_project | enum | Yes | `landing-page` or `desktop-app` |
| reference_style | string | No | Description or screenshot of desired visual direction |
| existing_file | path | No | Path to an existing component to restyle |
| shadcn_base | string | No | Which shadcn/ui component(s) to extend (e.g., `card`, `button`) |

## Step-by-Step Process

1. **Audit Existing State** — If restyling, read the current component. Identify vibe-code anti-patterns:
   - Default gradients or generic backgrounds?
   - Uniform grid with no visual hierarchy?
   - Unstyled shadcn defaults?
   - Icons used as decoration rather than communication?

2. **Define Visual Direction** — Before writing code, document 3-5 specific design decisions:
   - Color palette (from Hedge Edge brand tokens)
   - Spacing rhythm (varied, not uniform)
   - Typography hierarchy (font sizes, weights, letter-spacing)
   - Motion intent (what animates and why)
   - Edge cases (empty state, loading, error)

3. **Build Component Structure** — Use shadcn/ui as the structural base:
   - Import from `@/components/ui/<component>`
   - Extend with `cva()` (class-variance-authority) for custom variants
   - Use `cn()` helper for conditional Tailwind merging
   - Reference: [shadcn-component-patterns.md](../../tmp/shadcn-component-patterns.md)

4. **Apply Anti-Vibe-Code Styling** — Transform defaults:
   - Replace default borders with subtle `ring-1 ring-white/5` or brand-colored accents
   - Use `backdrop-blur` for glass effects instead of flat solid backgrounds
   - Apply asymmetric padding (`pt-8 pb-12` not `py-10`)
   - Add `transition-all duration-300` with custom easing for interactive states
   - Use Lucide icons at `strokeWidth={1.5}` with `size={16-20}` (never larger unless hero treatment)

5. **Add Iconography** — Select Lucide icons:
   - Browse: https://lucide.dev/icons
   - Import: `import { IconName } from "lucide-react"`
   - Every icon must answer: "What does this communicate to the user?"
   - Reference: [lucide-icon-guide.md](../../tmp/lucide-icon-guide.md)

6. **Validate** — Check against the Definition of Done

## Expected Output

- A React component file (`.tsx`) with TypeScript types
- Uses shadcn/ui primitives extended with custom variants
- Lucide icons used semantically (not decoratively)
- Custom Tailwind classes (not just default shadcn styles)
- Responsive design with dark-mode-first approach
- Component passes the anti-vibe-code checklist

## Definition of Done

- [ ] Component renders correctly in the target project
- [ ] No default shadcn/ui styling visible — all extended with brand tokens
- [ ] Icons are purposeful (every icon answers "what does this communicate?")
- [ ] Spacing is asymmetric and intentional (not uniform `p-4` everywhere)
- [ ] Color use is brand-consistent (dark-mode financial aesthetic)
- [ ] Interactive states (hover, focus, active) are visually distinct and animated
- [ ] Component is responsive across viewport sizes
- [ ] Passes the Anti-Vibe-Code Checklist (see resources)
- [ ] TypeScript types are defined for all props

## Error Handling

| Error | Cause | Resolution |
|-------|-------|------------|
| Component looks generic after styling | Still relying on shadcn defaults | Add custom `cva()` variants with brand colors, custom borders, and animation |
| Icons feel decorative | Wrong icon chosen or icon adds no information | Remove icon or replace with one that communicates specific meaning |
| Layout feels "template-y" | Symmetric grid, uniform spacing | Break symmetry — use bento grids, overlapping elements, staggered sizing |
| Tailwind classes getting very long | Over-styling in JSX | Extract to `cva()` variants or Tailwind `@apply` in component-level styles |
